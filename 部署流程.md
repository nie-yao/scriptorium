###### 1. 服务器准备

Ubuntu 上先装基础环境：

```shell
sudo apt install -y python3 nginx git curl
```

安装uv

```shell
curl -LsSf https://astral.sh/uv/install.sh | sh
source ~/.bashrc
```

如果还没开防火墙，可以顺手放行：

```shell
sudo ufw allow OpenSSH sudo ufw allow 'Nginx Full' sudo ufw enable 
```

###### 2. 上传项目

从 Git 仓库克隆代码：

```shell
git clone <你的仓库地址> cd your-service 
```

###### 3. 创建虚拟环境并安装依赖

同步：

```shell
uv sync
npm install
```

###### 4. 先手动启动测试

在服务器上用 uvicorn 启动：

```shell
source .venv/bin/activate uvicorn app:app --host 127.0.0.1 --port 8000 
```

然后在服务器上本机测：

```shell
curl http://127.0.0.1:8000/api/health 
```

如果返回正常，说明应用没问题。

###### 5. 配置 systemd 常驻运行

创建服务文件：

```shell
sudo vi /etc/systemd/system/reference-format.service 
```

填入：

```ini
[Unit] 
Description=[Your Service Name] FastAPI App 
After=network.target 
[Service] 
User=username 
Group=www-data 
WorkingDirectory=/home/username/your-service
Environment="PATH=/home/username/your-service/.venv/bin" 
ExecStart=/home/username/your-service/.venv/bin/uvicorn app:app --host 127.0.0.1 --port 8000 Restart=always RestartSec=3 [Install] 
WantedBy=multi-user.target 
```

把里面的 username 改成你的 Ubuntu 用户名。

然后启用：

```shell
sudo systemctl daemon-reload 
sudo systemctl enable your-service 
sudo systemctl start your-service 
sudo systemctl status your-service 
```

看日志：

```shell
journalctl -u your-service -f 
```

###### 6. 配置 nginx

新建 nginx 配置：

```shell
sudo vi /etc/nginx/sites-available/your-service 
```

填入：

```nginx
server {    
  listen 80;    
  server_name YourIP;     
  client_max_body_size 20M;     
  location / {        
    proxy_pass http://127.0.0.1:8000;        
    proxy_http_version 1.1;        
    proxy_set_header Host $host;        
    proxy_set_header X-Real-IP $remote_addr;        
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;        
    proxy_set_header X-Forwarded-Proto $scheme;    
  } 
} 
```

启用配置：

```shell
sudo ln -s /etc/nginx/sites-available/your-service /etc/nginx/sites-enabled/ 
sudo nginx -t sudo systemctl reload nginx 
```

这样你就可以通过：

```shell
http://你的服务器IP 
```

访问。

###### 7. 如果有域名，配置 HTTPS

推荐直接用 Certbot：

```shell
sudo apt install -y certbot python3-certbot-nginx sudo certbot --nginx -d your-domain.com 
```

它会自动改 nginx 配置。

###### 8. 更新服务

先拉取更新

```shell
git pull
```

再重启系统服务

```
sudo systemctl start your-service
```
